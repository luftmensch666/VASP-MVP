# 项目结构说明

本文档旨在详细描述 VASP-MVP 项目的文件和目录结构。项目旨在搭建一个 Debian 12 本地 VASP 工作流 MVP，包含前端页面、后端配置、安全边界设定、文件处理和数据流管理。

## 核心目录与文件详细说明

### `app.py`
- **角色**: Streamlit Web UI 的主入口文件。
- **职责**: 负责页面流程的串联与展示。包含结构上传、任务类型选择、自动生成草稿内容的预览、用户操作确认、启动任务、实时输出日志 (`vasp.out` 和 `OSZICAR`) 以及结果汇总页面的展示。遵循“只调用业务模块，不直接拼接路径、不直接跑 `shell=True`”的安全原则。

### `src/vasp_mvp/`
包含项目的核心业务模块与数据流控制逻辑，主要分为以下几个子模块：
- **`models.py`**: 定义核心数据模型（`dataclass`）。如：任务请求(`TaskRequest`)、任务草稿(`TaskDraft`)、运行指标(`Metrics`)、配置模型(`AppConfig`) 等。
- **`config.py`**: 配置读取管理。读取 `config/defaults.json` 和 `config/potcar_map.json`，集中管理白名单路径、可用线程数等环境参数。
- **`security.py`**: 系统的安全边界限制。强制所有路径位于 `workspace` 或 `POTCAR` 白名单内，防止路径逃逸；执行命令禁止 `shell=True` 并强制校验 MPI ranks 设定。
- **`structure_io.py`**: 结构文件的读取和转换模块。通过 ASE 解析 POSCAR/CIF 等格式，统一转换为 VASP5 标准格式并提取体系的元素、原子数等基本信息。
- **`rules.py`**: 任务规则生成器。根据不同的任务类型（`relax`, `static`, `molecule`, `adsorption`）生成配套的 INCAR 和 KPOINTS 参数，不做真实的文件写入。
- **`renderers.py`**: 负责将 `rules.py` 的结构化参数渲染成对应的文本字符串，例如 INCAR、KPOINTS、POTCAR 的安全合并命令（仅用于 UI 预览）和 `run.sh` 脚本的内容。
- **`db.py`**: 使用 SQLite 实现轻量级任务和指标数据的持久化存储，包含对 `tasks` 表和 `metrics` 表的 CRUD 交互操作。
- **`runner.py`**: 负责任务执行相关的逻辑。包括用户确认后创建和写入任务运行目录、构建安全无 shell 的 `subprocess.Popen` 参数列表以调用 VASP，并提供尾随读取运行日志的方法。
- **`parser.py`**: OUTCAR 和 OSZICAR 解析模块，用于提取能量（TOTEN）、平均循环时间、收敛状态等核心计算指标，处理可能出现的不收敛或文件缺失等异常状态。
- **`adsorption*.py`系列**: 吸附任务专属模块，负责复杂的 `E_ads` 吸附能的逻辑运算，涉及 slab、molecule 等多种结构处理（向导）和多模块之间的协调调用。
- **`vaspkit_*.py`系列**: 对外置第三方生成器 VASPKIT 的支持与交互控制（执行、解析），用于从外部扩展结构生成与输入文件生成。

### `config/`
- **`defaults.json`**: 定义 VASP 命令路径、工作空间目录及并行计算线程池和允许配置数。
- **`potcar_map.json`**: 将元素名称映射为实际的赝势文件相对路径目录名，由后端解析合并出完整的赝势合成命令。
- **`i18n/`**: 包含中文 (`zh.json`) 和英文 (`en.json`) 词典文件，通过 key-value 的形式实现前端语言界面的切换和文本多态。

### `docs/`
- 项目的文档目录。包括项目开发的初始设定（如 codex 对话记录）以及最终自动生成的说明手册等，是了解项目需求变更与设定的主要源泉。

### `tests/`
- **角色**: Python 单元测试集，采用标准库 `unittest` 框架。
- **职责**: 用于保障各个数据处理、配置校验和任务分发流程中的稳定性与安全性（如 mock subprocess 判断参数注入、测试路径逃逸策略等），确保迭代重构时不破坏旧有接口逻辑。

### 根目录配置文件
- **`requirements.txt`**: 列举 Python 第三方依赖。如 `streamlit`, `pandas`, `plotly`, `ase`。
- **`.gitignore`**: Git 忽略规则，特别是避免将计算中间文件、结果、`POTCAR` 本身（属于商业机密和体积庞大）及 `.venv` 追踪到库中。
- **`AGENTS.md`**: AI 和研发指南说明，制定了该 MVP 项目开发的一系列明确的底线安全规范与建议（如 POTCAR 下载、`shell=True` 禁止等）。
